# 估值宝 FundWatchWeb（基金净值/估值查看 · 自用）

纯前端 H5 应用，无需后端、无需上架。打开即用，能"添加到桌面"当 App 用。
专为 **华为 Pura X（HarmonyOS 6.1.0，纯血鸿蒙）** 适配——标准安卓 APK 在纯血鸿蒙上装不上，而本网页版在鸿蒙浏览器里直接跑。

> ⚠️ 数据说明（务必先看）
> 天天基金 `fundgz.1234567.com.cn` 的"盘中实时估值"JSONP **已于 2026 年被官方停用**（现 301 跳 notfound）。
> 本版**稳定数据源**（均跨域脚本/JSONP 注入，纯前端无后端）为：
> - **东方财富 `pingzhongdata.js`** → 基金名 + **最新确认净值 + 每日净值历史**（走势图，所有基金可用）
> - **腾讯 `qt.gtimg.cn/q=<mkt><code>`** → **场内基金（ETF / LOF）的实时行情**：当前价 / 昨收 = 盘中实时涨跌幅（**实时市价，精确，非估算**），交易时段自动显示。
> - **新浪 `stock.finance.sina.com.cn/fundInfo/api/openapi.php/FdFundService.getEstimateNetworthPic?symbol={code}`**（JSONP）→ **场外开放基金的盘中实时估值**：返回含今日时间戳的逐分钟估算净值序列（实测 2026-07 仍可用），交易时段自动显示为"盘中估值"，非交易时段回落到最新确认净值。该接口含两个估算口径，差异较大时详情页并列提示。
> 即：**场外基金交易时段看新浪盘中估值、其余看最新确认净值；场内基金（代码段 15/16/50/51/58 自动识别，详情页可手动开关）看腾讯实时价**。估值/实时价 ≠ 官方净值，仅供参考。

## 一、在 Pura X 上怎么用（三种方式，任选）

### 方式 A：最简单 —— 直接打开（推荐先试）
1. 把整个 `FundWatchWeb/` 文件夹拷到手机（微信文件传输/USB/网盘均可）。
2. 用**浏览器**打开里面的 `index.html`。
3. 浏览器菜单 → **"添加到桌面"**（鸿蒙浏览器一般叫"添加至桌面/收藏到桌面"）。
4. 以后点桌面图标就是 App。
> 注：部分鸿蒙浏览器在 `file://` 下可能**不保存数据**（localStorage 受限）。若发现刷新后自选丢失，改用方式 B/C。

### 方式 B：最稳（数据能保存，推荐）
在**电脑**上进入 `FundWatchWeb/` 目录，起一个本地服务器：
```bash
# 需 Python（mac/Win 装了 Python 即可）
python -m http.server 8080
```
然后让手机和电脑在**同一 WiFi**，手机浏览器访问：
```
http://<电脑局域网IP>:8080/index.html
```
例：`http://192.168.1.20:8080/index.html`。这样 localStorage 正常，数据持久保存。
（看电脑 IP：`ipconfig` / `ifconfig`）

### 方式 C：最像原生 App（PWA，可离线）
把 `FundWatchWeb/` 整个文件夹部署到任意**静态托管**（GitHub Pages / Vercel / 腾讯云静态网站 等，免费），用 https 域名访问，再"添加到桌面"即可获得类原生体验（含离线缓存）。

## 二、功能

- **自选首页**：分组标签栏（全部/自定义分组）+ 基金卡片（净值/实时价 + 涨跌% 红涨绿跌 + 所属分组 + 持仓收益）。**场内基金（ETF/LOF）交易时段显示"实时"徽标与盘中实时价**
- **添加基金**：搜索代码/名称 → 每行可指定加入哪个分组 → 一键添加
- **基金详情**：大号净值 + 涨跌 + **净值历史走势图**（SVG）+ 持仓设置（份额/成本价）→ 自动算市值与收益
- **持仓收益**：总览（总市值/总成本/总收益）+ 逐只明细
- **分组管理**：新建 / 重命名 / 删除分组；**删除分组时组内基金会自动回到"未分组"，不会被删**
- **Pura X 折叠屏适配**：屏幕展开（宽屏 ≥840px）自动切成**左列表 + 右详情**双栏；外屏/竖屏自动回单栏
- **三态兜底**：加载中（骨架屏）、空（引导添加）、错误（缓存兜底，绝不白屏）
- **免责常驻**：界面内标注"估值 ≠ 官方净值"

## 三、文件结构

```
FundWatchWeb/
├── index.html            页面骨架（多文件版，方式 A/B/C 用）
├── styles.css           设计系统（红涨绿跌、响应式、双栏）
├── app.js              全部逻辑：数据层(JSONP) + 状态 + 5 个界面渲染 + Pura X 适配
├── manifest.webmanifest PWA 清单（添加到桌面用）
├── sw.js               Service Worker（同源资源离线缓存）
├── FundWatchWeb-single.html  ★ 单文件版：CSS+JS 全内联，一个文件搞定（分享用）
├── build-single.js     单文件构建脚本（改完源码后 `node build-single.js` 重新生成）
└── README.md          本说明
```

> 💡 **单文件版 `FundWatchWeb-single.html`**：把 `styles.css` 和 `app.js` 全部内联进一个 HTML，
> 没有任何外部依赖（不引用 manifest/sw.js）。适合**直接发给朋友**——对方用浏览器打开即可，
> 同样支持添加基金、持仓收益、分组、红涨绿跌、折叠屏双栏。
> 注意：单文件版**不含"添加到桌面/离线"的 PWA 能力**（那需要独立的 manifest+sw.js 文件），
> 若想要桌面图标/离线，请用多文件版挂到静态托管（方式 C）。

## 四、技术要点

- 数据获取全部走 `<script>` 注入（JSONP 思路），**天然跨域、无需 CORS/Referer**，鸿蒙浏览器同样适用。
- 净值刷新为顺序请求（每只基金依次拉取），避免多脚本全局变量互相覆盖。
- 持仓与分组存 `localStorage`；取数失败有超时（9s）与缓存兜底。

## 五、已知限制

1. **场外基金盘中估值已接新浪**：交易时段用新浪 `FdFundService` 估值曲线显示盘中实时估值（逐分钟、含今日时间戳）；非交易时段回落到最新确认净值。**场内基金（ETF / LOF）用腾讯实时行情显示盘中实时价**——交易所实时市价，比估算更准。识别靠代码段（15/16/50/51/58），添加时自动判定，详情页也可手动开关。
2. **QDII / 海外基金**净值公布有 T+1~T+7 延迟，属正常现象，非 bug。
3. 仅自用，请勿对外分发或商用（数据来自第三方公开接口，遵守其使用条款）。

## 六、单文件分享版（给朋友用）

`FundWatchWeb-single.html` 是**一个独立的 HTML 文件**（CSS/JS 全内联，无外链）。

**怎么发给朋友 / 朋友怎么用：**
- 直接通过微信/邮件/网盘把 `FundWatchWeb-single.html` 这一个文件发过去；
- 朋友**用手机/电脑浏览器打开它**即可使用（无需任何安装、无需上架）；
- 想当 App，浏览器菜单选"添加到桌面"即可（部分浏览器对单文件 `file://` 的本地保存有限制，数据可能不持久；要稳定保存可改用下方方式）。

**分享时数据说明：**
- 单文件版和朋友是**各自独立**的——朋友添加的基金、填的持仓只存在他自己的浏览器里，不会和你互通（本应用无任何后端/账号）。
- 朋友电脑若装了 Python，也能用方式 B 起本地服务器获得持久保存。

**重新生成单文件：** 当你改了 `app.js`/`styles.css` 后，在工程目录运行 `node build-single.js` 即可重新产出 `FundWatchWeb-single.html`（以源文件为准，避免两边不一致）。
